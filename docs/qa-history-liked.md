# QA — 대화 히스토리 & 저장된 응답

---

## 검증 범위

| 영역 | 확인 내용 |
|------|-----------|
| 좋아요 저장 | 따봉 클릭 → DB 영구 저장 여부 |
| 저장된 응답 패널 | 패널에 좋아요 메시지가 표시되는지 |
| 히스토리 전송 | 이전 대화 내용이 백엔드로 전달되는지 |
| 히스토리 활용 | AI가 이전 맥락을 기억하며 응답하는지 |

## 1. 좋아요 저장 & 저장된 응답 패널

### TC-01 따봉 클릭 시 DB 저장 확인

**전제조건:** 대화가 1개 이상 존재하고 assistant 응답이 있을 것

**절차:**
1. 채팅창에서 아무 메시지나 전송
2. AI 응답이 완료되면 응답 하단의 👍 버튼 클릭
3. 브라우저 개발자도구 → Application → IndexedDB → MilkywayDB → messages 열기
4. 해당 메시지 레코드의 `liked` 값 확인

**기대 결과:**
- `liked: true`, `pinned: true` 로 저장되어 있을 것
- 👍 버튼이 파란색(활성화)으로 표시될 것

**실패 시 확인 포인트:**
- `chatRepository.setMessageLiked()` 가 호출되고 있는지 → `ChatMessageItem`의 `handleFeedback` 로그 확인
- DB version이 5로 마이그레이션되었는지 → IndexedDB 패널에서 `messages` 스토어에 `liked` 인덱스 존재 여부 확인

---

### TC-02 따봉 취소 시 DB 업데이트 확인

**절차:**
1. TC-01 완료 후, 활성화된 👍 버튼을 다시 클릭 (토글 취소)
2. IndexedDB → messages 에서 해당 레코드 재확인

**기대 결과:**
- `liked: null`, `pinned: false` 로 변경될 것
- 👍 버튼이 비활성화 색상으로 복귀할 것

---

### TC-03 👎 클릭 시 DB 저장 확인

**절차:**
1. AI 응답 하단의 👎 버튼 클릭
2. IndexedDB → messages 해당 레코드 확인

**기대 결과:**
- `liked: false`, `pinned: false` 로 저장될 것
- 👎 버튼이 파란색(활성화)으로 표시될 것

---

### TC-04 저장된 응답 패널 — 좋아요 메시지 표시

**절차:**
1. TC-01 완료 상태 (따봉 누른 응답 존재)
2. 사이드바 하단 **저장된 응답** 버튼 클릭
3. 패널이 열리는지 확인
4. 따봉 누른 응답 내용이 패널에 표시되는지 확인

**기대 결과:**
- 패널이 Sheet(드로어) 형태로 열릴 것
- 따봉 누른 assistant 메시지 내용이 카드 형태로 표시될 것
- 날짜, [이동] 버튼, [삭제] 버튼이 각 카드에 존재할 것

**실패 시 확인 포인트:**
- `LikedMessagesPanel`의 `useEffect`에서 `chatRepository.getLikedMessages()` 호출 여부
- `getLikedMessages()`의 `.equals(1)` 조건 — Dexie에서 boolean `true`는 숫자 `1`로 저장됨, `.equals(true)` 사용 시 조회 안 될 수 있음

---

### TC-05 저장된 응답 패널 — [이동] 버튼

**절차:**
1. 저장된 응답 패널에서 카드의 [이동] 버튼 클릭

**기대 결과:**
- 패널이 닫힐 것
- 해당 응답이 속한 대화방으로 전환될 것 (사이드바에서 해당 대화가 선택됨)

---

### TC-06 저장된 응답 패널 — [삭제] 버튼 (좋아요 취소)

**절차:**
1. 저장된 응답 패널에서 카드의 [삭제] 버튼 클릭

**기대 결과:**
- 패널에서 해당 카드가 즉시 사라질 것
- IndexedDB → messages 해당 레코드의 `liked` 값이 `null`로 변경될 것
- 채팅창으로 돌아가면 해당 응답의 👍 버튼이 비활성화 상태일 것

---

### TC-07 페이지 새로고침 후 상태 유지

**절차:**
1. TC-01 완료 후 브라우저 새로고침 (F5)
2. 해당 대화방으로 진입
3. 따봉 누른 응답의 👍 버튼 상태 확인
4. 저장된 응답 패널에서 메시지 표시 확인

**기대 결과:**
- 새로고침 후에도 👍 버튼이 파란색(활성화) 상태 유지
- 저장된 응답 패널에도 동일한 메시지가 표시될 것

## 2. 메시지 히스토리 전송 확인

### TC-08 히스토리가 백엔드로 전달되는지 네트워크 확인

**절차:**
1. 새 대화 시작
2. 첫 번째 메시지: "내 이름은 철수야"
3. AI 응답 대기
4. 두 번째 메시지: "내 이름이 뭔지 알아?"
5. 두 번째 메시지 전송 시 브라우저 개발자도구 → Network → `/api/v1/chat` POST 요청 클릭 → Payload 탭 확인

**기대 결과:**
```json
{
  "message": "내 이름이 뭔지 알아?",
  "history": [
    { "role": "user", "content": "내 이름은 철수야" },
    { "role": "model", "content": "..." }
  ]
}
```
- `history` 배열에 첫 번째 턴의 user/model 메시지가 포함될 것

**실패 시 확인 포인트:**
- `useChat.ts`의 `buildHistory(storedMessages)` 호출 여부
- `streamChat` 함수에 `history` 파라미터가 전달되는지
- `api/chat.ts`에서 `body.history`가 설정되는지

---

### TC-09 AI가 이전 대화 맥락을 기억하는지 확인

**절차:**
1. 새 대화 시작
2. 첫 번째 메시지: "나는 파이썬 개발자야. 앞으로 코드 예시는 파이썬으로만 보여줘."
3. AI 응답 대기
4. 두 번째 메시지: "리스트에서 중복을 제거하는 방법 알려줘"

**기대 결과:**
- AI가 파이썬 코드 예시(`set()`, list comprehension 등)로 응답할 것
- JavaScript나 다른 언어 예시가 나오지 않을 것

**히스토리 없을 때와의 비교 (선택):**
- 새 대화를 열어 히스토리 없이 "리스트에서 중복을 제거하는 방법 알려줘"만 보내면 언어 선택이 없거나 다를 수 있음

---

### TC-10 멀티 턴 맥락 유지 확인

**절차:**
1. 새 대화 시작
2. 1턴: "1부터 10까지 더하면 얼마야?"
3. AI 응답 확인 (55 또는 계산 과정)
4. 2턴: "거기에 100을 더하면?"
5. AI 응답 확인

**기대 결과:**
- AI가 "앞서 계산한 55에 100을 더하면 155입니다"처럼 이전 계산 결과를 이어받을 것
- "무엇을 100에 더하라는 건지 모르겠습니다" 같은 맥락 단절 응답이 나오지 않을 것

---

### TC-11 좋아요 메시지가 히스토리에 고정 포함되는지 확인

**목적:** 대화가 길어져 최근 20개 제한을 초과해도, 좋아요 누른 메시지가 히스토리에 포함되는지 검증

**절차:**
1. 새 대화에서 첫 번째 메시지 전송 후 AI 응답에 👍 클릭 (고정 대상 지정)
2. 이후 10턴 이상 다른 주제로 대화 진행
3. Network 탭에서 마지막 전송의 `history` payload 확인

**기대 결과:**
- `history` 배열에 1번째 턴의 user/model 메시지가 포함될 것 (좋아요로 고정됨)
- 최근 20개 외 오래된 메시지는 포함되지 않을 것

## 3. 자동 요약 (대화 제목 갱신)

### TC-12 3턴(6개 메시지) 도달 시 제목 자동 갱신

**절차:**
1. 새 대화 시작 (초기 제목: "새 대화")
2. 3번의 메시지 교환 진행 (user 3개 + assistant 3개 = 총 6개)
3. 6번째 메시지(3번째 AI 응답) 완료 후 사이드바 확인

**기대 결과:**
- 사이드바의 대화 제목이 "새 대화"에서 대화 내용을 요약한 한 문장(30자 이내)으로 변경될 것
- 예: "파이썬 리스트 중복 제거 방법 질의응답"

**실패 시 확인 포인트:**
- Network 탭에서 `POST /api/v1/chat/summarize` 요청이 발생했는지 확인
- `useChat.ts`의 `storedMessages.length + 2 === 6` 조건이 트리거되었는지

## 실패 시 공통 체크리스트

```
[ ] IndexedDB MilkywayDB의 messages 스토어에 liked 인덱스가 존재하는가
    → Application > IndexedDB > MilkywayDB > messages > Indexes 확인

[ ] DB version이 5인가
    → Application > IndexedDB > MilkywayDB 에서 version 확인

[ ] /api/v1/chat POST body에 history 필드가 포함되는가
    → Network > /chat > Payload 확인

[ ] /api/v1/chat/summarize 엔드포인트가 존재하는가
    → 3턴 후 Network 탭에서 요청 발생 여부 확인

[ ] 좋아요 클릭 시 UI 즉시 반응하는가 (DB 저장과 별개)
    → 버튼 색상 변화 확인

[ ] 새로고침 후 좋아요 상태가 복원되는가
    → F5 후 해당 메시지의 버튼 상태 확인
```
